# AGENTS.md — opencode-config

## Stack
- Go 1.23 + Wails v2.12 backend
- React 18 + Vite 3 frontend (plain JS/JSX, **not TypeScript**)
- No testing, linting, or CI configured

## Commands
```sh
wails dev     # Live development (HMR + Go hot-reload)
wails build   # Production build
```

Run these from the project root. The frontend can also be worked on independently from `frontend/` via `npm run dev` / `npm run build`, but `wails dev` is the intended workflow.

## Architecture
- **Go entrypoint**: `main.go` — embeds `frontend/dist` via `//go:embed`, binds `App` to JS
- **Frontend entrypoint**: `frontend/src/main.jsx`
- **`App` struct** (`app.go`): exported methods become callable from JS
- **Wails JS bindings**: `frontend/wailsjs/go/main/App` — auto-generated by `wails generate module`
- **Browser dev** (without desktop window): `http://localhost:34115` (Go methods accessible from devtools)
- **Config**: `wails.json` controls app name (`OpenCode Config`), output filename (`opencode-config.exe`), and frontend scripts

## Gotchas
- `frontend/wailsjs/` is auto-generated — do not edit by hand; regenerate with `wails generate module`
- `go.mod` has a local `replace` directive for the Wails module — it's pointing to a local Go cache path
- All Go files belong to `package main` (flat package)
- The frontend `wailsjs/` import path is relative: `../wailsjs/go/main/App`
